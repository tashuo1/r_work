# 本体数据平台产品功能清单（一期）

> 版本：v1  
> 日期：2026-05-20  
> 范围声明：仅覆盖第一期（3个源业务库验证 + T+1 日更 + 问数闭环）。

## 0. 当前代码落地口径（2026-05-22）

> 本节用于区分“当前可运行轻量版”和“生产化一期目标”。除非明确标注为生产化目标，当前验收以 `ontology-lite/ontology-server` 实际代码为准。

当前可运行版本：

1. 运行形态：单进程 `Spring Boot 3 + MyBatis`，静态产品界面由同一服务托管，访问地址 `http://127.0.0.1:9000/`。
2. 持久化：平台运行态优先通过 MyBatis 写入 MySQL/PolarDB 表 `ontology_meta.om_platform_state`，保存数据源、计算资源、数据标准、模型版本、ETL、问数、审计、T+1 调度等 JSON 快照；外部平台库不可达时启动脚本自动切换到本地 H2 兜底，保证一期产品页面可演示；计算资源中的 `META` / `INSTANCE` 数据库连接信息同步落入结构化 `ontology_meta.om_compute_resource / om_compute_resource_health`，账号和密码保存供后续使用但接口与日志不展示密码；旧 `ontology_platform_state` 已废弃删除，不再自动创建或兼容读取。
3. 数据源：页面采用类 DataWorks 的列表页 + 新增数据源弹层 + 创建/编辑 DM 数据源表单，列表行仅保留编辑/删除，连接信息列隐藏 JDBC 查询参数并支持换行，表格按容器自适应且不显示底部横向滑动条；支持财务/投资/人事三类源库连接信息录入、密码保存状态展示、真实 JDBC 连接测试、删除数据源、内部启停和页面手动巡检；连接参数由用户填写，MySQL/PolarDB 连接串自动补齐连接/读超时参数，前端 API 网络异常显示中文诊断；驱动类和默认参数由后台展示或按类型推导，前台表单不展示；认证文件管理一期不提供入口。
4. 建模：当前支持工作台式四子页（元数据扫描、候选建模、语义图谱、版本发布）、Schema-Only 演示扫描、扫描失败快照降级提示、候选生成中即时反馈、候选模型生成、语义图谱节点/关系详情联动、影响评估、页面内发布确认弹层、发布和回滚；真实 `ontology-modeling` Skill 稍后接入。
5. 数据标准：当前已将原“标准词典”升级为数据标准页面，包含 `AI生成 / 命名词根 / 标准字段` 三个子页；AI生成支持多选数据源并按 `cizhu-shengcheng` 输出结构生成 `DRAFT` 候选，命名词根和标准字段支持人工确认/拒绝，`REJECTED` 默认隐藏，确认后同步 active 计算资源的标准四表；当前仍保留 CSV 导出，Excel 模板导入导出一期先不做。
6. T+1：当前支持调度查看/时点修改、手动触发、质量结果和 watermark 展示；尚未执行真实源库抽取和实例表 upsert。
7. 问数：当前支持自然语言触发、受控 SQL、安全闸、图表数据和 CSV 内容导出；SQL 执行结果来自演示实例数据，不是实际数据库查询。
8. API：当前内部接口和 OpenAPI 均为本地无鉴权接口；OAuth2.1/OIDC、限流、幂等窗口、审计保留 90 天属于后续生产化治理目标，一期先不做。
9. 产品界面：01-09 主功能菜单支持折叠/展开，折叠后保留编号入口并通过浏览器本地存储记住用户上次选择。

## 0. 功能分级决策记录（逐轮确认）

1. 数据连接能力：**B（增强版）**  
   - 手工录入连接信息、连接测试、启用/停用  
   - 连接模板  
   - 页面手动连通性测试  
   - 后台定时巡检与失败告警后续再做
2. 本体论建模能力：**C（进阶版目标）**  
   - 当前已有 Skill，能力现状约在 B 阶段  
   - 一期目标：在现有 Skill 基础上优化到 C，支持多轮对话修正与自动优化闭环  
   - 输入方式：自然语言 + 模板混合（参考 ontology-modeling skill）  
   - 生效原则保持不变：模型输出需经过人工确认后生效
3. 语义修改确认流程：**A（单人确认）**  
   - 一期采用建模人直接确认生效  
   - 确认备注非必填  
   - 支持回滚最近生效版本（单次最多10个）  
   - 生效前提供影响评估提示，需二次确认  
   - 二期升级为流程管控（复核/审批链路）
4. 数据标准：**新增独立页面（一期纳入）**  
   - 页面包含 `AI生成 / 命名词根 / 标准字段` 三个子页  
   - AI生成支持多选数据源，按 `cizhu-shengcheng` 输出结构生成待审核候选；当前轻量版用确定性候选生成器模拟 Skill 输出，真实 LLM/Skill 执行器后续替换  
   - 命名词根和标准字段可查看、可审核、可更新  
   - 编码规则：词根采用单一中文业务词的汉语拼音；标准字段支持已登记规范词根的下划线组合  
   - 候选治理：支持 `ACTIVE` / `DRAFT` / `REJECTED`，`DRAFT` 默认展示为待审核，`REJECTED` 默认隐藏，候选证据通过 `source_type`、`confidence_level`、`evidence_json`、`review_note` 承载  
   - 相近词根：在命名词根卡片中展示相近词根及关系，能匹配到目标词根时写入 `om_root_synonym`  
   - 生效时机：数据标准更新在下次建模任务生效  
   - 支持导出：当前代码为 CSV；Excel 一期先不做  
   - 设计参考 DataWorks 的词根与标准字段治理思路
5. 本体语义展示：**C（双视图联动）**  
   - 关系图 + 表格并存  
   - 支持双向联动定位（图点表、表点图）
6. 本体实例抽取策略：**B + C 结合（并行多线程）**  
   - 核心对象采用 C（稳定优先）  
   - 非核心对象采用 B（性能优先）  
   - 抽取任务采用多线程并发执行（每库初始并发=4，带并发上限与失败回补）  
7. 本体论问数能力：**C（增强交互）**  
   - 支持多轮追问（上下文继承，保留5轮）  
   - 支持 SQL 可视化编辑后重跑  
   - 支持分级结果解释模板（摘要/明细/口径）
8. API 接口能力：**C（内外API + 动作触发）**  
   - 内部 API：供前端、调度、治理模块调用  
   - 对外 API：只读查询与元数据读取  
   - 动作触发 API：触发建模、同步、问数任务
   - 鉴权：生产化目标为 OAuth2.1/OIDC Bearer JWT，一期先不做  
   - 限流：生产化目标为租户+接口双维限流，IP限流兜底，一期先不做  
   - 幂等：动作触发接口保留 `requestId` 字段，一期不做 24h 窗口强制去重  
   - 回查超时：60s；审计日志保留90天

## 2. 功能清单（按模块）

### 2.1 数据源管理

1. 新增数据源（一期固定为达梦三库）。  
2. 编辑/删除数据源；内部接口仍保留启停能力，列表行不直接展示启停入口。  
3. 连通性测试。  
4. 只读权限校验。  
5. 页面手动连通性测试与状态展示。  
6. 后台 30 分钟巡检和红色异常感叹号后续再做。  
7. 源库并发保护（单库并发上限=10）。
8. 页面交互参考 DataWorks 数据源管理：数据源列表、类型筛选、名称筛选、新增数据源弹层和创建/编辑 DM 数据源表单；列表行仅展示“编辑/删除”，JDBC 列表展示隐藏 `?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true&connectTimeout=10000&socketTimeout=10000` 等查询参数并支持长地址换行，表格不出现底部横向滑动条；驱动类、默认参数、高级参数不在前台表单展示，认证文件管理入口已删除，一期不做。

**一期验收点：**
- 当前轻量版：支持在页面维护 3 个测试源的 JDBC URL、主机、端口、库/模式名、账号和密码，并执行真实 JDBC 连接测试；驱动类和默认参数由后台默认处理，JDBC 查询参数在列表展示中隐藏；MySQL/PolarDB 默认生成和用户手工填写的 JDBC URL 均会补齐 `connectTimeout=10000` 与 `socketTimeout=10000`。  
- 当前一期目标：用户填入财务/投资/人事三套测试源连接信息后，三库连接测试能给出真实成功或失败结果。  
- 用户体验要求：本地服务未启动、服务不可达或请求超时时，页面必须显示中文诊断，不得裸露 `Failed to fetch`。  
- 后台 30 分钟定时巡检和异常红色感叹号不作为当前一期门禁。  
- 当前页面不提供认证文件管理，新增入口仅保留数据源类型选择与连接信息填写。

### 2.2 语义建模（本体语义层）

1. 元数据扫描（表/字段/主外键/注释/索引）。  
2. 自动生成对象/关系/属性候选。  
3. 建模结果可视化展示（候选对象/属性/关系列表 + 语义图谱 + 右侧详情面板）。  
4. 人工确认、修改、删除、新增。  
5. 生效发布与审计日志。  
6. 置信度分档机制（十分制）：高置信度阈值 `>= 9.0`。  
7. 多轮修正指令支持：重命名、合并、删除、重映射。  
8. 自动优化策略：仅生成建议，不自动批量执行。  
9. 驳回原因分类记录（用于后续 Skill 学习优化）。  

**一期验收点：**
- 核心对象建模完成并生效；人工确认通过率 >= 70%。  

### 2.2.2 本体语义展示（C 双视图联动）

1. 语义图谱：对象节点、关系连线、业务域分组和节点/关系点击详情。  
2. 候选建模页：对象/属性/关系清单，展示来源表、标准字段映射、置信度和建模建议。  
3. 联动能力：
   - 图上点击对象，高亮并定位表格对应记录；  
   - 表格点击记录，关系图自动聚焦到对应节点/关系。  
4. 版本视图：仅展示“当前生效语义版本”，支持切换查看历史失效版本。  
5. 差异视图（可选增强）：展示“候选模型 vs 生效模型”差异点。  

**一期验收点：**
- 双视图联动稳定可用，且在核心对象范围内定位准确。  

### 2.2.1 语义确认与发布（一期 A 方案）

2. 生效动作记录审计日志（操作人、时间、变更内容）。

### 2.7 数据标准管理（新增）

1. AI生成页：多选数据源后生成命名词根和标准字段候选，返回生成任务、Skill 名称、数据源数量、候选数量和处理状态。  
2. 命名词根页：词根编码（单一中文业务词的汉语拼音）、词根中文名、业务定义、业务域、来源、置信等级、证据、审核说明、状态；默认展示 `ACTIVE` 与 `DRAFT`，可勾选查看 `REJECTED`。  
3. 标准字段页：字段编码、字段名称、数据类型、口径说明、关联词根、来源、置信等级、证据、审核说明、状态；默认展示 `ACTIVE` 与 `DRAFT`，可勾选查看 `REJECTED`。  
4. 标准字段拼接键：支持独立存储（如 `00001_yonghu_mingcheng`）。  
5. 词根与标准字段映射维护（多对多）。  
6. 相近词根展示：命名词根卡片展示相近词根、关系类型、置信权重和说明，匹配后同步到 `om_root_synonym`。  
7. 审核流转：`DRAFT` 候选支持确认为 `ACTIVE` 或拒绝为 `REJECTED`；确认后同步 active 计算资源的 `om_term_root / om_standard_field / om_root_field_map / om_root_synonym`。  
8. 导出（当前代码支持 CSV；Excel 模板导入导出一期先不做）。  
9. 导入失败支持错误明细下载（逐行错误定位）。  
10. 数据标准变更生效时机：下次建模任务生效。  
11. 搜索与筛选（按业务域、状态、更新时间）。  
12. 更新记录审计（谁修改、修改前后差异）。  
13. 建模 Skill 引用数据标准能力（提示词上下文增强）。  
14. 辅助识别词根只用于缩略/历史简称识别，不得组成正式标准字段。  

**一期验收点：**
- 数据标准页面可访问，并包含 `AI生成 / 命名词根 / 标准字段` 三个子页。  
- AI生成能基于多选数据源生成待审核候选，候选带来源、置信、证据和审核说明。  
- 命名词根与标准字段均可查看、确认、拒绝和更新，并可被建模 Skill 正常引用。  
- 相近词根能在命名词根列表中展示，并通过 `om_root_synonym` 表达与主词根的关联。  
- 词根编码符合单一汉语拼音词规范；标准字段能完整拆回已登记规范词根；多对多映射可维护。  
- 当前轻量版 CSV 导出可用；Excel 导入导出一期先不做。  
- 数据标准更新在下次建模候选生成中体现。  

### 2.3 实例更新（T+1）

1. T+1 定时任务运行（默认 `01:00`，支持用户手动改时点）。  
2. A/B/C 增量策略配置（按表）。  
3. `watermark` 记录与断点续跑。  
4. 幂等 `upsert` 写入实例层。  
5. 失败重跑与质量门禁。  
6. 多线程并行抽取与装载（按对象/表分片并行）。  
7. 并发保护机制（线程池上限、源库限流、库连接池保护）。  
8. 回补任务机制（失败分片自动重试3次，仍失败则人工补跑）。  

#### 2.3.1 B+C 组合执行口径

1. 核心对象（财务/投资/人事核心对象）：
   - 默认采用 C 策略（稳定优先），优先保证口径一致与成功率。  
2. 非核心对象：
   - 默认采用 B 策略（性能优先），提高吞吐并允许小范围回补。  
3. 并行执行原则：
   - 同库内分片并行 + 跨库并行；  
   - 对热点大表设置独立并发上限，避免压垮源库。  
   - 每库初始并发线程数为 4，按监控再调优。  
4. 调优策略：
   - 先低并发试运行，再按监控逐步提并发。  
   - 以“成功率、时延、源库负载”三指标联合调优。  

**一期验收点：**
- 连续运行期同步成功率 >= 98%。  
- 并行执行状态下，源库未出现不可接受的性能抖动。  
- 质量门禁阈值生效：行数波动20%、重复率0%（按物理主键）；空值率仅监控提示。  

### 2.4 调度与编排（方案A）

1. 根据生效语义层自动生成对象/关系任务。  
2. 任务依赖自动编排（对象 -> 关系 -> 质量 -> 发布）。  
3. 默认生产时点 `01:00`，支持用户手动配置。  
4. 如需优化，支持多时点试运行并收敛正式时点。  

**一期验收点：**
- 默认 `01:00` 调度稳定可用，且手动改时点后任务可正常执行。  

### 2.5 智能问数（执行在实例层）

1. 自然语言问题输入。  
2. 语义层辅助 SQL 生成。  
3. SQL 安全闸（只读、LIMIT、超时、EXPLAIN）。  
4. 在实例层执行 SQL 并返回结果。  
5. 结果解释与图表展示。  
6. 多轮追问（继承上轮语义上下文与筛选条件）。  
7. SQL 可视化编辑并重跑（保留改写前后版本）。  
8. 允许跨库查询，但仅限本体实例层白名单schema。  
9. 禁止直接跨源业务库在线join。  
10. 图表类型支持用户切换。  
11. 问数结果下载当前支持 CSV 内容导出；Excel 下载一期先不做。  
12. 解释模板分级：
   - 摘要层：结论与关键指标；  
   - 明细层：分组结果与明细表；  
   - 口径层：统计口径、时间范围、过滤条件说明。  

**一期验收点：**
- 10 条核心问句可稳定执行并返回可解释结果。  
- 多轮追问链路可用，SQL编辑重跑可用，解释分级输出完整。  
- 跨库查询仅在实例层白名单内生效；跨源库在线join被拦截。  
- 图表切换与 CSV 下载可用；Excel 下载一期先不做。  

### 2.6 审计与可观测

1. 任务运行日志（run_log）。  
2. 增量水位日志（watermark）。  
3. 建模变更审计（谁、何时、改了什么）。  
4. 告警通知（失败、超时、行数突变、重复键）。  

**一期验收点：**
- 关键任务可追溯，异常可定位，失败可重跑。  

### 2.8 API 接口能力（C：内外API + 动作触发）

1. 内部管理 API：
   - 数据源管理（新增/编辑/启停/测试）  
   - 语义建模（扫描、候选生成、确认生效）  
   - 实例同步（任务触发、重跑、状态查询）  
2. 对外只读 API：
   - 语义元数据查询（对象/关系/属性/词根/标准字段）  
   - 本体实例查询（按对象、按条件分页）  
   - 问数结果查询（任务结果、SQL、解释）  
3. 动作触发 API：
   - 触发建模任务（按源库/业务域）  
   - 触发 T+1 同步任务（按批次/对象范围）  
   - 触发问数任务（异步执行并回查结果）  
4. 通用治理能力（生产化目标，一期先不做）：
   - 鉴权（OAuth2.1/OIDC Bearer JWT）  
   - 限流（按租户+接口双维，IP兜底）  
   - 幂等（动作触发接口 `request_id` 必填）  
   - 异步回查默认超时（60s）  
   - 审计日志保留（90天）  
   - 审计（请求人、参数摘要、耗时、结果）  

**一期验收点：**
- 当前轻量版：核心内部 API、对外只读 API、动作触发 API 本地可用，并记录业务变更审计。  
- 生产化目标：接口通过 OAuth2.1/OIDC 鉴权，幂等、限流、超时、审计保留策略按配置生效；当前一期先不做。  

## 3. 一期不做清单（防范围蔓延）

1. 不做实时 CDC。  
2. 不做 3 库之外的大规模扩库接入。  
3. 不做方案B（代码生成 + 编译部署执行）。  
4. 不做复杂跨域高级分析与自动决策闭环。  
5. **不做数据内容分析驱动的建模**（一期仅做 Schema-Only 建模，不做字段值采样/状态机推断/值级关联推断，这些能力在 `om_mapping_rule` 和 `om_action_type` 表中预留扩展字段，二期启用）。  
6. **不做全量自动化建模**（一期坚持"候选生成 + 人工确认"的半自动模式，不追求无人值守的全自动建模生效）。  
7. 不做 Excel 导入导出，当前只交付 CSV。
8. 不做 API 鉴权、限流、24h 幂等窗口和生产化审计保留策略。
9. 不做后台 30 分钟数据源巡检和告警推送，当前只做页面手动测试。

## 9. 研发无脑执行与自测清单（DoD）

### 9.1 每个功能开发卡必须包含

1. 功能描述（对应模块编号，如 `2.3`）。  
2. 输入输出定义（接口入参、出参、错误码）。  
3. 数据表影响（新增/修改字段、索引、约束）。  
4. 配置项清单（默认值、环境差异）。  
5. 自测用例ID（必须引用第7章）。  
6. 回滚方案（配置回滚、数据回滚、任务回滚）。  

### 9.2 提测前强制自测门禁

1. 本功能关联 `P0/P1` 用例执行完成。  
2. 单元测试通过，关键分支覆盖（成功/失败/边界）。  
3. 当前一期验证本地 API 可用；接口鉴权、限流、生产化审计日志后续再做。  
4. 数据质量门禁验证通过（行数、重复键、空值异常）。  
5. 变更说明已同步到本功能清单。  

### 9.3 联调阶段强制门禁

1. 三库联调至少完成一次全链路跑通。  
2. T+1任务完成“成功一次 + 失败重跑一次 + 回补一次”。  
3. 问数完成“单轮 + 多轮 + SQL编辑重跑”三类联调。  
4. API完成“内部触发 + 对外只读 + 审计追踪”联调。  

### 9.4 上线验收门禁

1. 第7章 `P0` 用例 100% 通过。  
2. 第7章 `P1` 用例 >= 90% 通过。  
3. 连续7天任务稳定性达标（成功率 >= 98%）。  
4. 里程碑 M5 全部必过用例完成签字。  
5. 已完成灰度发布并通过内测库验证。  
6. 已完成回滚演练，满足30分钟恢复SLA。  
7. 通过验收后，立项进入P1能力补齐迭代。
