## 0.0 当前代码事实与已确认执行口径（2026-05-22）

当前代码已经从旧基座收敛为 `Spring Boot 3 + MyBatis` 分层单体，并通过 `ontology_meta.om_platform_state` 持久化平台运行态。运行配置从 `ontology-lite/runtime-config/ontology-platform.env` 读取；当前平台持久化库使用 MySQL/PolarDB，H2 仅作为无配置时的本地兜底。

2026-05-22 已确认的当前一期执行口径：

1. **三库连接测试必须接入真实信息**：数据源页面必须支持录入财务/投资/人事三套测试库连接信息，并执行真实 JDBC 连接测试；测试信息由用户在页面填写。
2. **建模 Skill 稍后接入**：当前候选模型仍由服务内置规则生成，真实 `ontology-modeling` Skill 调用不作为当前这一轮阻塞项。
3. **平台持久化使用 MySQL/PolarDB**：平台运行态先继续采用 `ontology_meta.om_platform_state` JSON 快照表持久化；计算资源中的 `META` / `INSTANCE` 数据库连接信息同步落入 `ontology_meta.om_compute_resource / om_compute_resource_health`，账号和密码保存供后续使用但接口与日志不展示密码；旧 `ontology_platform_state` 已废弃删除，不再自动创建或兼容读取；`om_platform_state` 不与 `om_data_source` 混用。
4. **Excel 一期先不做**：问数结果和词典导出当前以 CSV 为准，Excel 导入/导出不作为当前一期验收项。
5. **API 治理一期先不做**：OAuth2.1/OIDC、租户限流、24h 幂等窗口、审计保留 90 天保留为生产化目标，不作为当前一期门禁。
6. **后台巡检告警一期先不做**：数据源支持页面手动测试和状态展示，30 分钟后台巡检与红色异常感叹号不作为当前一期门禁。

## 0. 当前已锁定且不再重复讨论项（2026-05-20更新）

1. 一期范围：3个达梦源库 + T+1 + 问数闭环。  
2. T+1 默认 `01:00`，支持手动改时点。  
3. 并发：每库初始 4，源库上限 10。  
4. 质量阈值：行数波动 20%、重复率 0%、空值率仅监控。  
5. API 生产化目标：OAuth2.1/OIDC + JWT；`request_id` 幂等；回查 60s；审计 90 天。当前一期先不做 API 治理实现，仅保留本地接口形态。  
6. 词典规则：词根为单一中文业务词的汉语拼音；标准字段为已登记规范词根的下划线组合；多对多映射；辅助识别词根只用于 `om_root_synonym`，不得参与标准字段组成；候选治理通过四表内 `status/source_type/confidence_level/evidence_json/review_note` 承载。
7. 元数据中心库一期选型：MySQL。  
8. 调度引擎一期选型：轻量版内置调度服务。  
9. P1 用例通过率门槛：>=90%。  
10. 一期核心对象范围：不冻结，按迭代动态维护。  
11. `SYSDBA` 当前不设截止时间，但数据源必须支持随时修改连接信息。  
12. 重复率0%判定口径：按物理主键。  
13. 生效影响评估固定字段：影响对象数/影响关系数/影响任务数。  
14. 词典导入失败支持错误明细下载。
15. **PM-02（P0）**：一期不允许临时插入新业务域对象，必须走变更评审流程（产品发起→架构评估→研发排期）。
16. **ARC-02（P0）**：SYSDBA 当前不收敛，M5 后评估创建专用只读账号 `ontology_readonly`。
17. **ARC-03（P0）**：跨库白名单 schema 由平台管理员在数据源管理页面维护，`om_data_source` 表增加 `allowed_schemas_json` 字段。
18. **TEC-02（P0）**：SQL 安全闸规则已冻结（仅SELECT/自动注入LIMIT+timeout/EXPLAIN预检/禁止跨源库join/白名单拦截），见 `4_技术规范（接口+数据库）.md` 第一部分第3节。
19. **SEC-01（P0）**：token 签发优先对接企业统一 IAM（OAuth2.1/OIDC）；兜底内置 JWT 签发（secret 轮换 180 天），二期强制切换 IAM。
