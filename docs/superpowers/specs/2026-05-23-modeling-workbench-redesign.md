# 本体建模任务导向工作台设计

日期：2026-05-23

## 背景

当前本体建模页已经覆盖元数据扫描、候选建模、语义图谱、版本发布四个子功能，但首屏信息同权展示，用户进入页面后难以判断当前模型处于哪个阶段、下一步应该做什么、哪些事项存在风险。

本次重塑仅面向 PC 网页端，不考虑移动端。

## 目标用户

- 数据架构师：确认对象、属性、关系是否可以进入语义层。
- 数据开发人员：执行扫描、生成候选、发布版本和回滚。
- 产品/验收人员：快速判断本体建模链路是否可演示、可验收。

## 产品目标

进入页面 5 秒内回答四个问题：

1. 当前模型处于什么阶段。
2. 下一步最该执行哪个动作。
3. 是否存在阻塞或风险。
4. 如果要看细节，应进入哪个步骤。

## 设计方案

采用“任务导向建模驾驶舱”结构：

- 状态总览区：显示当前阶段、当前候选版本、生效版本、确认率、模型资产和最近输入。
- 下一步动作区：仅突出一个主动作，其余动作降级为次操作。
- 风险提示区：集中展示 DRAFT 版本堆积、确认率不足、扫描未完成等需要注意的事项。
- 流程步骤区：用元数据扫描、候选确认、语义图谱、版本发布四步表达任务进度，而不是纯功能分类。
- 聚焦工作区：默认进入最需要处理的步骤。存在 DRAFT 候选时优先显示发布/候选确认；没有候选时显示扫描。

## 非目标

- 不新增后端接口。
- 不改变现有建模、发布、回滚 API 行为。
- 不接入真实 `ontology-modeling` Skill。
- 不做移动端专项适配。

## 验收标准

- PC 端 `#modeling` 首屏显示“当前阶段”和“下一步动作”。
- 主按钮只有一个视觉重点。
- 页面能明确展示生效版本、候选版本、确认率、对象/属性/关系数量。
- 存在多个 DRAFT 版本时给出风险提示。
- 元数据扫描、候选确认、语义图谱、版本发布仍可通过步骤切换访问。
- 扫描、生成候选、确认生效、回滚主流程按钮仍可点击。
- PC 截图无明显文字重叠、按钮拥挤和横向溢出。
